### 问题2: "项目中遇到过什么困难?"

推荐案例:

```
困难1: Socket连接状态管理

问题: 用户网络波动时Socket可能中断，导致消息丢失和界面不一致。

解决方案:
1. 实现自动重连机制(指数退避算法)
2. 消息队列本地缓存，重连后重发
3. 用户界面实时反馈连接状态
4. 后端存储消息，用户上线时同步历史

结果: 系统可靠性提升到99.9%

---

困难2: 10万+并发连接支持

问题: 初期内存泄漏，100并发时内存溢出。

分析: Socket事件监听未清理，大量闭包占用内存。

解决方案:
1. 用WeakMap存储用户连接
2. 正确清理事件监听(disconnect)
3. 批量消息处理而非逐条
4. 连接池管理

结果: 支持从100扩展到10万+并发
```

### "为什么选择这些技术?"

**推荐回答**:

```

选择理由:

React: 社区大,组件化思想匹配,热更新开发体验好
Socket.io: 比原生WebSocket更稳定,自动降级处理,
事件模型更清晰
Express: 轻量级,适合快速原型,中间件系统灵活
TypeScript: 编译期提前发现错误,团队协作时类型文档作用大
Vite: 开发速度快,HMR体验好

取舍:
如果重新选择,考虑:

- 状态管理: 当前简单用Props,大型应用可用Redux
- 数据库: 当前内存存储,生产需PostgreSQL
- 部署: Docker容器化,Kubernetes编排

```
